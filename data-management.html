Project Introduction

3 Data management

During one day at an hypothetical internship, an e-mail was sent to me by a colleague, asking me to analyse some data from a plate reader experiment where wells were measured over time. The data is provided in an Excel file attached to the e-mail. The colleague asks me to generate a plot per sample, no other information was provided. Listed below are the steps that I learned to apply in a situation like this.

3.1 Checking the data’s contents

Before loading any type of data into R/RStudio, it’s contents and metadata should be checked first. This include format type, checking for NA’s, what datasheet should be imported (when working with Excel files).

3.2 Storing the data

After this step the data can be downloaded and stored in a raw_data folder located in the appropriate repository, preferably through code. Metadata, if provided, will be stored in a README.txt file also located in the raw_data folder. If old versions of the freshly downloaded files are present in the raw_data folder, they will need to be moved to a new subfolder V1 (or V2, V3… depending on the version number).

3.3 Loading the data into R

The now locally stored data can be loaded into R using the appropriate packages. For example, for loading Excel files the {readxl} package can be used. To increase robustness, relative pathing should be used. This can be done with the {here} package.

3.4 Data wrangling

Whit the data now loaded in R, data wrangling can begin (if necessary). Important things to consider at this stage is which packages will be used later during the analysis and what type of data formatting this requires. For example: ggplot2 is one of the packages which I worked with a lot during my studies, which requires the data to be in the tidy format.
